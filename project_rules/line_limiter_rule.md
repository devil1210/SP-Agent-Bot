# Regla de Límite de Líneas por Archivo

## Regla
Cada archivo de código no debe exceder las 1000 líneas de código.
Idealmente, los archivos deben mantenerse por debajo de 500 líneas.

## Acciones
- Si un archivo excede 500 líneas, considerar refactorizarlo
- Si un archivo excede 1000 líneas, refactorizar obligatoriamente
- Dividir archivos grandes en módulos más pequeños con responsabilidades únicas
- Usar principios de Single Responsibility Principle (SRP)

## Beneficios
- Código más legible y mantenible
- Fácil depuración y testing
- Mejor colaboración en equipo
- Reduce la complejidad del código
